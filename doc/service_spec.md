# サービス契約システム

## サービス仕様
### 概要
ユーザから申込を受け、サービス管理者から承認されたらAPIキーを払い出しを行う。  
ただしTrialについては承認不要とする。

### プラン
* Trial
  * 基本利用料：0
  * トークン利用料：100万トークンあたり100円
  * トークン使用上限：10000/日
  * 利用可能モデル：miniのみ
* Starter
  * 基本利用料：20000円/月
  * トークン利用料：100万トークンあたり1000円
  * トークン使用上限：500000/日
  * 利用可能モデル：すべて
* Pro：
  * 基本利用料：30000円/月
  * トークン利用料：100万トークンあたり1000円
  * トークン使用上限：1000000/日
  * 利用可能モデル：すべて


### 申込要項
ユーザはフォームから申込を行う。  
* プロジェクトID
* APIキー名
* プラン
* 利用目的
  * PCで実行：研究
  * PCで実行：作業効率化
  * ITシステムへ組込：開発/検証
  * ITシステムへ組込：本番
  * その他

### 料金計算
毎月2日に先月分の料金を算出する。  
料金は「基本利用料」と「トークン利用料」から構成される。
基本利用料の日割りは行わない。  


### 課金請求
「予算番号」に対して、請求明細を作成する。
請求明細は1つの「見出し」と、1つ以上の「品目」「金額」「備考」を持つ。

### プロジェクト
プロジェクトは以下の項目を持つ。
* プロジェクトID
* プロジェクト名
* 予算番号
* 予算番号の管理責任者
* 連絡先
* メンバー

チームはシステム外のマスターデータで管理される。



## ビジネスユースケース
### 新規：サービス利用申込
### 変更：プラン変更
### 終了：解約申込

予算番号などが変更された場合は、マスターデータ側が直接更新される。


## 契約状態
### 申込中
まだ申込しただけで、APIキーの払い出しには至っていない状態。
この状態では料金は発生しない。

### 利用中
APIキーの払い出しが完了し、利用できるようになった状態。  
毎月利用料金が請求される。

### 利用終了済
ユーザから解約が申込まれた状態。
APIキーは利用できなくなる。
